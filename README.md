https://github.com/Xeney/student-projects-hub

**Цель проекта:** Получение практических навыков командной веб-разработки, работы с Git,
взаимодействия frontend и backend, а также углубление знаний в веб-технологиях.

**В результате выполнения проекта команда должна:**
1. Создать полноценный веб-сайт с разделением frontend и backend
2. Реализовать клиент-серверное взаимодействие по HTTP
3. Освоить основы совместной работы с Git/GitHub
4. Получить проект, который можно использовать в портфолио.

*Проект должен быть работоспособным, логически завершённым и иметь понятную структуру.*

## Концепция проекта
**Название:** Student Projects Hub.
**Описание:** Веб-приложение для размещения и просмотра учебных проектов.

**Пользователь может:**
1. Просматривать список проектов
2. Открывать страницу конкретного проекта
3. Добавлять новый проект
4. Редактировать существующий проект
5. Удалять проект.

## Распределение ролей

**Frontend-разработчик**
**Зона ответственности:**
1. Разработка дизайна интерфейса в Figma
2. Вёрстка страниц (HTML, CSS)
3. Клиентская логика на JavaScript
4. Взаимодействие с backend через API
5. Адаптивность интерфейса.

**Backend-разработчик**
**Зона ответственности:**
1. Разработка серверной части на Golang
2. Проектирование API
3. Реализация бизнес-логики
4. Работа с хранилищем данных
5. Обеспечение корректной обработки запросов.

## Функциональные требования

### Frontend
**Страницы сайта** 
1. **Главная страница**
	1. краткое описание проекта
	2. кнопка перехода к списку проектов.
2. **Страница списка проектов**
	1. отображение всех проектов в виде карточек
	2. каждая карточка содержит
		1. название проекта
		2. автора
		3. краткое описание
		4. кнопка открытия проекта
		5. кнопка удаления проекта
3. **Страница проекта**
	1. полная информация о проекте
		1. название
		2. описание
		3. автор
		4. дата создания
		5. кнопка редактирования проекта
4. **Страница добавления / редактирования проекта**
	1. форма с полями
		1. название проекта
		2. описание
		3. автор
		4. базовая валидация (поля не пустые)
		5. кнопка сохранения

### Backend
**Проект представляет собой объект со следующими полями:**
```json
{
	"id": 1,
	"title": "Tetris на C#",
	"description": "Консольная версия игры",
	"author": "Дима",
	"created_at": "2026-01-15"
}
```

**API эндпоинты**

| Метод  | URL              | Описание                 |
| ------ | ---------------- | ------------------------ |
| GET    | /api/projects    | Получить список проектов |
| GET    | /api/projects/id | Получить проект по ID    |
| POST   | /api/projects    | Создать новый проект     |
| PUT    | /api/projects/id | Обновить проект          |
| DELETE | /api/projects/id | Удалить проект           |
*Формат обмена данными - JSON*

## Критерии готовности проекта

**Проект считается завершённым, если:**
1. Реализованы все CRUD-операции
2. Frontend и Backend корректно взаимодействуют
3. Используется база данных
4. Присутствует базовая валидация данных
5. Проект запускается по инструкции из README
6. Git-история читаемая и логичная.
